android相关-移动端组件化

Jan 7, 2016 | android | Hits

###android

##移动端组件化，以独立功能单元为独立组件，组件即服务的方式提供给使用方。

初步预计现有组件结构，可分为三层，在后续实现过程中，可能会根据具体情况做相应调整

#通用层(common层):提供通用能力，例如Log日志、json解析、通用加密、task管理和其他常用工具。

#基础服务层：提供移动端开发的所有常用基础能力。包含多个组件，例如：DB、cache、http、image、page、plugin、hotfix。
DB 提供通用数据库能力，对数据库进行封装管理，提供给外部使用
cache 提供数据缓存能力，可以为http、image等服务提供数据缓存管理
http 提供http请求管理及数据解析能力，可为image、plugin、hotfix等提供数据加载能力
image 提供图片加载能力
page 提供页面跳转管理能力
plugin 提供插件化服务，使项目插件化
hotfix 热修复能力，提供线上修复部分bug的能力

#通用组件层：提供各种高复用性的通用组件，可分为四类，特殊业务类、通用UI类、设计规范类、通用能力类，例如：

#特殊业务类：
OSS图片上传 提供基础的OSS图片上传能力
Octopus 提供客户端数据抓取能力
埋点统计 提供通用埋点能力
支付能力 提供微信和支付宝支付能力
活体识别 提供接入活体识别的能力
通用UI类：通用UI为没有出公司设计规范的常用自定义UI，如果有相应规范，可调整到设计规范类
StateView 提供页面状态的统一交互，例如：加载中、加载失败、空页面、加载成功等状态
RefreshView 提供页面刷新的统一交互
Dialog 提供弹出框的自定义统一交互
ImagePicker 提供统一的图片选择、剪裁等交互能力
Titlebar 提供统一的Titlebar交互能力
设计规范类：
通用Item样式 提供通用的表单ITEM样式
录入Item样式 提供录入能力，例如车辆录入
Toast样式 提供公司规范的toast样式
通用能力类：
webview 提供自定义与h5交互协议的通用webview
沉浸式通知栏 提供android平台的沉浸式通知栏交互
自定义字体 提供Android平台自定义app使用字体的能力
性能监测 提供app性能监测，为app性能问题提供帮助
消息push 提供消息push的能力，依赖后台交互协议的规范统一
